🟢 **基礎（全員）** ／ 🔧 **応用（読み飛ばしOK）** の目印つき。同じカテゴリの中は、だいたい出てくる順に並べています。

## 基本のことば

- 🟢 **CI（シーアイ）** … 継続的インテグレーション（Continuous Integration）。コードを上げる**たびに、自動でテスト／ビルドして「壊れてないか」を確認**するしくみ。→[01](01-what-is-cicd.md)
- 🟢 **CD（シーディー）** … 継続的デリバリー／デプロイ（Continuous Delivery / Deployment）。CIが通ったら**自動で届ける（デプロイ・リリース）**こと。→[01](01-what-is-cicd.md)
- 🟢 **パイプライン** … コミットから出荷まで**順番に流れる一連の自動処理**。工場の🏭ベルトコンベアそのもの。→[01](01-what-is-cicd.md)
- 🟢 **GitHub Actions（ギットハブ・アクションズ）** … GitHubに組み込まれた**「○○したら△△する」自動化のしくみ**。追加費用なしで始められる（無料枠あり）。→[02](02-github-actions.md)

## しくみのことば

- 🟢 **ワークフロー（workflow）** … 自動でやる作業の**手順書（YAMLファイル1つ）**。`.github/workflows/` に置く。`on:`（いつ）と `jobs:`（やること）でできている。→[02](02-github-actions.md)
- 🟢 **ジョブ（job）** … ワークフローの中の**ひとまとまりの仕事**。複数のジョブは**並行**して動かせる。→[02](02-github-actions.md)
- 🟢 **ステップ（step）** … ジョブの中の**1手順**。`uses:`（既製の部品を使う）か `run:`（コマンドを実行）。→[02](02-github-actions.md)
- 🟢 **ランナー（runner）** … 処理を動かす**GitHubが貸してくれる使い捨てのパソコン**。`runs-on:` で指定（例 `ubuntu-latest`）。→[02](02-github-actions.md)
- 🟢 **トリガー（`on:`）** … ワークフローを動かす**きっかけ**。「pushされたら」「PRが来たら」など。→[02](02-github-actions.md)
- 🟢 **アクション（action）** … 「ソースを取ってくる」などの**再利用できる既製の部品**（🧩レゴ）。`uses:` で呼ぶ。→[02](02-github-actions.md)
- 🟢 **Marketplace（マーケットプレイス）** … 公開アクションが**数千そろう品ぞろえの棚**。ここから部品を選んで使う。→[02](02-github-actions.md)
- 🟢 **YAML（ヤムル）** … ワークフローを書く**設定ファイルの書き方**。インデント（字下げ）で意味が決まる。→[03](03-workflow-yaml.md)

## トリガー（きっかけ）のことば

- 🟢 **push（プッシュ）** … コードを**プッシュしたとき**に動かす、いちばん基本のきっかけ。→[02](02-github-actions.md)
- 🟢 **pull_request（プルリクエスト）** … **PRが来たとき**に動かす。マージ前のチェックに使う。→[02](02-github-actions.md)
- 🔧 **schedule / cron（スケジュール／クロン）** … **決まった時刻に定期実行**（毎朝バッチ等）。`cron` で時刻を指定する。→[02](02-github-actions.md)
- 🔧 **workflow_dispatch（ワークフロー・ディスパッチ）** … GitHubの画面の**手動実行ボタン**で動かす。→[02](02-github-actions.md)
- 🔧 **tag / release（タグ／リリース）** … **タグを付けた／リリースしたとき**に動かす。リリース自動化に使う。→[04](04-catalog.md)

## 保存・速度のことば

- 🟢 **アーティファクト（artifact）** … ワークフローが作った**成果物**（ビルド済みアプリ・テストレポート等）を**保存・受け渡し・ダウンロード**できる。キャッシュとは別物（こちらは成果物を残す）。→[03](03-workflow-yaml.md)
- 🔧 **キャッシュ（cache）** … 依存（`node_modules` 等）を**保存して次回を速く**するしくみ。成果物ではなく速度のための一時保存。→[06](06-speed-cost.md)
- 🔧 **マトリクス（matrix）** … **複数のOS／バージョンを並行**でテストするしくみ（例 ubuntu/macos/windows × Node 18/20）。→[06](06-speed-cost.md)

## 安全のことば

- 🟢 **シークレット（secrets）** … APIキーなどの鍵を預ける🗝**金庫**。`${{ secrets.NAME }}` で参照し、**コードには書かない**。→[05](05-secrets-security.md)
- 🔧 **GITHUB_TOKEN（ギットハブ・トークン）** … ワークフローに**自動で渡る一時トークン（使い捨ての鍵）**。自分で作らなくてよい。→[05](05-secrets-security.md)
- 🔧 **permissions（パーミッション／最小権限）** … トークンにできることを**必要な分だけに絞る**設定。事故の被害を小さくする。→[05](05-secrets-security.md)
- 🔧 **pin（ピン／SHA固定）** … 他人のアクションを**特定の中身（SHA）に固定**して、勝手に中身が変わらないようにすること。信頼の担保。→[05](05-secrets-security.md)
- 🔧 **fork PR（フォーク・プルリクエスト）** … 他人のコピー（fork）から来るPR。**シークレットを渡さない**のが既定（`pull_request_target` は危険なので注意）。→[05](05-secrets-security.md)

## 自動化の実例ことば

- 🟢 **CodeQL（コードキューエル）** … GitHubの**セキュリティスキャン**。コードの**脆弱性（バグの穴）を自動で見つける**。パブリックリポジトリは無料。→[04](04-catalog.md)
- 🟢 **Dependabot（ディペンダボット）** … **依存ライブラリの更新**や**脆弱性アラート**を自動でPRにしてくれる係。→[04](04-catalog.md)
- 🟢 **レビューBot（レビューボット）** … PRに自動でレビューコメントを付ける係（CodeRabbit / Sourcery / Copilot code review 等）。人のレビュー前の一次チェック。→[04](04-catalog.md)
- 🟢 **デプロイ（deploy）** … 作ったものを**本番に届ける**こと。push→自動デプロイ（Vercel/Netlify/GitHub Pages 等）。→[04](04-catalog.md)
- 🟢 **リリース（release）** … **タグをpushしたらGitHub Releaseを自動作成**し、アーティファクトを添付する自動化。→[04](04-catalog.md)
- 🔧 **self-hosted runner（セルフホステッド・ランナー）** … **自前のマシンをランナーにする**しくみ。無料の計算力・自由が手に入る。→[06](06-speed-cost.md)
- 🔧 **required check（リクワイアド・チェック／必須チェック）** … **これが通らないとマージできない**と決めた自動チェック。壊れたコードの混入を防ぐ門番。→[04](04-catalog.md)
