# 付録A　各LLMのAPIキーの取り方（OpenAI / Anthropic）

> 📖 このページのゴール：自分のAPIキーを発行し、使いすぎないよう上限も設定する。
> [← 目次・はじめにへもどる](README.md)

---

## 🤔 APIキーって？

**APIキー（えーぴーあいきー）**＝LLM（AIの頭脳）を使うための、**秘密の合鍵（あいかぎ）**です。この鍵を見せると「私はこのサービスを使う権利があります」と証明できます。

そして大事なのは、**使った分の料金は、鍵の持ち主（あなた）に請求される**ということ。だから鍵は **絶対に他人に渡してはいけません**（第2章でくわしく学びました）。このページでは、その鍵を実際に **発行（はっこう＝新しく作ること）** していきます。

---

## 🪜 OpenAIのキーを取る

OpenAI（ChatGPTを作っている会社）のキーは、次の手順で取れます。**上から順に**やってください。

1. ブラウザで **platform.openai.com** を開き、**サインイン**します（アカウントが無ければ「Sign up」で新規作成。Googleアカウントでも作れます）。
2. ログインできたら、画面のメニューから **「API keys」**（エーピーアイ キーズ）を開きます。
3. **「Create new secret key」**（新しい秘密の鍵を作る）ボタンを押します。名前を聞かれたら、自分が分かるもの（例：`chatgpt-clone`）を付けてOKです。
4. 鍵（`sk-` で始まる長い文字列）が表示されます。**この場で必ずコピーして、安全な場所に貼り付けておきます。** ⚠️ **この鍵は一度しか表示されません**（画面を閉じると二度と見られません）。
5. 鍵を使うには **支払いの設定** が必要です。メニューの **「Billing」**（ビリング＝課金）を開き、クレジットカードなどを登録します（未登録だと、呼んでもエラーになります）。
6. **必ず「Usage limits」（ユーセージ リミッツ＝使用上限）を設定します。** 「1か月でいくらまで」という上限を決めておくと、万一たくさん使われても、それ以上は請求されません。**初心者は、まず小さめの金額（例：5ドル）にしておく**のが安心です。

> ⚠️ 手順4のコピーを忘れた場合は、もう一度 **手順3**（Create new secret key）からやり直して、**新しい鍵を作り直します**（古い鍵は使わないので削除してOK）。

---

## 🪜 Anthropic（Claude）のキーを取る

Anthropic（Claudeを作っている会社）のキーも、流れはほぼ同じです。

1. ブラウザで **console.anthropic.com** を開き、**サインイン**します（無ければ新規作成）。
2. メニューから **「API Keys」**（エーピーアイ キーズ）を開きます。
3. **「Create Key」**（鍵を作る）ボタンを押し、名前を付けます。
4. 表示された鍵を、**その場でコピー**して安全な場所に保存します（OpenAIと同じく、**後から見直せないことが多い**ので注意）。
5. **使用上限（Usage limits）** を設定します。OpenAIと同じく、月の上限額を小さめに決めておきましょう。

> 💡 この教材の本編は **OpenAI（GPT）** を中心に進みます。Anthropic（Claude）は必須ではありませんが、**付録Cで「OpenAI ↔ Anthropic の切り替え」**を試すときに使うので、**今のうちに取っておくと、あとがスムーズ**です。

## 🔑 鍵の置き方（超重要）

取った鍵は、**コードに直接書かず**、`.env`（環境変数のファイル）に置きます。考え方は第2章とまったく同じです。

```bash
# .env  ← このファイルは絶対にgitに上げない
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
```

**1行ずつ読むと：**
- `OPENAI_API_KEY=sk-...`：OpenAIで取った鍵を、`OPENAI_API_KEY` という名前で置く。
- `ANTHROPIC_API_KEY=sk-ant-...`：Anthropicの鍵も、必要なら同じファイルに置く（使わないなら無くてOK）。

そして、この鍵入りファイルを **gitに上げない**ように、`.gitignore` に登録します。

```bash
# .gitignore
.env
```

**1行ずつ読むと：**
- `.env`：鍵の入ったファイルを **git管理から外す**（公開リポジトリに鍵を上げてしまう事故を防ぐ）。

守るべきことは、たった3つです。

- **gitに上げない**（`.gitignore` に `.env` を入れる）。
- **ブラウザに出さない**（鍵を読むのはサーバーだけ。`process.env` 経由で読む）。
- **漏れたら即・無効化して再発行する**。一度でもネットに出た鍵は「漏れた」とみなし、管理画面で削除して新しく作り直します。くわしくは [第2章 秘密のAPIキーを守る](02-secret-key.md) を読み返してください。

## ⚠️ よくあるつまずき

- **支払いを設定していないのに呼んで、`401` や `quota`（クォータ）エラーが出る** → OpenAIは Billing の登録が済むまで使えません。手順5を済ませましょう。
- **「無料枠（むりょうわく）があるはず」と思い込む** → 無料のお試し分は、無かったり、期限切れだったりします。**基本は「使った分だけ有料」**と考えるのが安全です。
- **使用上限を設定せずに公開して、使いすぎる** → 上限（Usage limits）を決めないと、青天井で請求が増える危険があります。**必ず小さめの上限から**始めましょう。
- **鍵は2回見られないのに、メモし忘れる** → 発行直後にコピーし忘れると、その鍵はもう取り出せません。落ち着いて作り直し（新しい鍵を発行）ましょう。

---

## 📝 ことばメモ

- **APIキー**：LLMを使うための秘密の合鍵。料金は持ち主に請求される
- **Usage limits（使用上限）**：「いくらまで使ってよいか」の上限。事故的な高額請求を防ぐ
- **課金（billing／ビリング）**：使った分の料金を支払う仕組み・その設定画面
